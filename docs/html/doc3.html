Conceptos importantes antes del manejo de Github

Antes de comenzar a revisar los comandos básicos de Github es importante tener presentes algunos conceptos básicos como lo son: repositorio en línea, repositorio local y un repositorio remoto . Al comenzar a utilizar github de forma colaborativa usualmente se nos pedirá hacer un fork al repositorio de un compañero y una vez hecha nuestra parte del trabajo se tendrá que relizar un pull request al compañero o a propietario del repositorio, todo esto se continuación se explicará con el fin de que la próxima vez que nos enfrentemos a esto sepamos que se debe hacer.

Repositorio
Es un espacio de almacenamiento en donde se guardan y acceden a archivos como por ejemplo algún proyecen el que se esté trabajando.
Para más información de click aquí

Repositorio en línea
Repositorio que está guardado en el servidor de github.

Repositorio local
Se trata de una copia de algún repositorio en línea dentro de nuestro ordenador.

Repositorio o remoto
Se trata del repositorio en línea de un tercero al cual hicimos un fork.
Para más información de click aquí

Fork
Crear una copia del repositorio en línea de otro perfil en nuestro perfil de github. Desde este repositorio, podemos colaborar y sugerir cambios en el repositorio original.

Pull request
Se trata de una solicitud de incluir cambios en el repositorio de un tercero realizados en un repositorio en línea surgido a partir de realizarle un fork.

Proceso para actualizar un repositorio remoto al que se hizo Fork

  • Hacemos fork al repositorio remoto con el que estaremos trabajando
  • Una vez que tenemos la "copia" en nuestro repositorio remoto, se debe clonar a nuestro repositorio local

    • $ git clone url

    donde el url es de nuestro repositorio remoto.

  • Teniendo la copia, dentro de nuetro repositorio local nos movemos a la carpeta del repositorio.
  • Si modificamos alguno de los archivos dentro de nuestro repositorio local, debemos agregarlos y comentarlos

    • $ git add [name_file]

    Después de agregarlo, guardamos o confirmamos los cambios con el siguiente comando:

    $ git commit -m '[comment]'

  • Actualizamos la copia del repositorio local nuestro fork
    • Utilizando el siguiente comando actualizamos la copia del repositorio original:

    $ git push

  • Generamos el Pull Request
    • Por último, una vez actualizada nuestra copia (el fork) del repositorio remoto, generamos la solicitud para actualizar el repositorio remoto original

    Proceso para actualizar nuestro repositorio local desde un repositorio remoto

  • Primero, una vez dentro de nuestro repositorio local, nos movemos a la rama que queremos actualizar con los cambios hechos por el propietario del repositorio remoto
    • Utilizando el siguiente comando nos movemos a la rama de interés

    $ git checkout RAMA

  • Actualizamos nuestro repositorio local desde el repositorio original pidiendo los cambios que se hayan hecho:

    • $ git remote add --track master NICKNAME url

    donde el NICKNAME un identificador que le daremos al repositorio original y éste puede ser cualquier nombre, el url es del repositorio remoto original.

    Ten en cuenta que este proceso es cuando nosotros no somos los propietarios del repositorio original. En dado caso de que quisieramos actulizar nuestro repositorio local siendo propietarios de un repositorio remoto el comando anterior es innecesario y simplemente tendríamos que escribir en la línea de comando:

    $ git pull origin RAMA

  • Si nosostros no somos propietarios del repositorio original con el comando pull comando "bajamos" los cambios

    • $ git pull NICKNAME RAMA

  • Finalmente actualizamos la respectiva rama en nuestro repositorio remoto

    • $ git push origin RAMA
